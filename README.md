A research project investigating **privacy risks in synthetic tabular data generated by diffusion models**.

The system implements a **Membership Inference Attack (MIA)** in a **black-box setting**, where an attacker attempts to determine whether a specific record was used during the training of a generative model.

This project evaluates whether **synthetic data generated by diffusion models leaks information about the original training data**.

---

# Project Motivation

Synthetic data is often shared as a **privacy-preserving alternative to real datasets**.

However, modern generative models can unintentionally **memorize training samples**, allowing attackers to infer whether specific records were part of the training dataset.

Membership inference attacks aim to answer the question:

> Was this record used to train the model?

Understanding these risks is essential when synthetic datasets are released for research or commercial use.

# Attack Pipeline Workflow

## 1. Dataset Preparation

The project uses datasets from the **MIDST Challenge**.

Each dataset contains:

• training dataset  
• synthetic dataset generated by diffusion models  
• challenge dataset for membership inference  

Dataset usage:

| Dataset Range | Purpose |
|---------------|--------|
| 1 – 20 | Train attack model |
| 21 – 30 | Evaluate attack |

---

## 2. Distance Feature Extraction

For each **challenge record**, distances to synthetic samples are computed.

Distance metrics used:

• Euclidean distance  
• Cosine distance  
• Mahalanobis distance  
• Maximum Mean Discrepancy (MMD)

These distances measure **similarity between challenge samples and synthetic data**.

---

## 3. Statistical Feature Engineering

From the computed distances, several statistical features are extracted:

• minimum distance  
• mean distance  
• variance  
• percentile statistics  

These features help identify **patterns indicating membership in the training data**.

---

## 4. Attack Model Training

Using datasets **1–20**, a classifier is trained to distinguish:

• **Members** – records used in training  
• **Non-members** – unseen records

The classifier learns patterns that indicate whether a sample influenced the generative model.

---

## 5. Attack Evaluation

The trained attack model is applied to **datasets 21–30**.

Evaluation metric used:

**TPR @ 10% FPR**

This metric measures the ability of the attack to correctly detect training samples while controlling false positives.
